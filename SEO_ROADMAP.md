# SEO Roadmap: Local AI Model Recommender

This roadmap defines a 25-page SEO content plan for the Local AI Model Recommender project.

Current stack:
- Static HTML/CSS/JS
- GitHub Pages
- Cloudflare Web Analytics
- Google Search Console
- No backend
- No build step

Primary CTA for every article: **Local AI Model Recommender**

## Publishing Rhythm

- Publish 2-3 pages per week.
- Submit each new page in Google Search Console after publishing.
- After 8-10 new pages are live, review Google Search Console for impressions, queries, click-through rate, and indexing status.
- Use early query data to refine titles, intros, FAQs, and internal links.

## Standard Article Template

Use this structure for each new SEO article:

1. Title / H1
   - Match the target query closely.
   - Keep it specific and beginner-friendly.

2. Short answer
   - Answer the main query within the first 100-150 words.
   - Include realistic hardware expectations when relevant.

3. Who this page is for
   - Define the reader's hardware, skill level, or use case.
   - Avoid repeating generic copy across pages.

4. Recommendation table
   - Include model names, RAM/VRAM expectations, best use case, speed expectation, and beginner notes.
   - Keep recommendations practical for local hardware.

5. Detailed recommendations
   - Explain each model or decision clearly.
   - Mention tradeoffs: speed, quality, RAM, VRAM, context length, and ease of use.

6. Setup or usage notes
   - Include simple commands only when relevant.
   - Link to beginner setup pages instead of duplicating full install instructions.

7. Common mistakes
   - Cover overlarge models, low RAM, slow CPU-only usage, missing storage, and unrealistic expectations.

8. FAQ
   - Add 4-6 concise questions based on the article's search intent.
   - Keep answers direct and non-duplicative.

9. CTA
   - End with a clear link to the Local AI Model Recommender.
   - Position it as the fastest way to get a model recommendation for the reader's hardware.

## Internal Linking Rules

- Every article should link to `index.html` with CTA-focused anchor text.
- Every article should link to at least 2 related articles in the same cluster.
- Every article should link to at least 1 article from another cluster when it naturally helps the reader.
- Use descriptive anchors such as "check whether Ollama can run on 8GB RAM" instead of generic "click here".
- Add links near the point where the reader needs the next answer, not only at the end.
- Link from completed high-relevance pages to new pages after each new article is published.
- Keep navigation useful for beginners: hardware pages should lead to model recommendation pages, and model pages should lead to setup/troubleshooting pages.

## Content Quality Rules

- Do not create thin duplicate pages.
- Each page needs a distinct search intent, audience, and recommendation angle.
- Reuse the article structure, but do not reuse the same intro, tables, FAQs, or conclusions.
- If two pages would answer the same query, combine them or make one page a supporting section instead.
- Prioritize practical recommendations over keyword repetition.
- Update recommendations when model availability, Ollama behavior, or common hardware expectations change.

## Search Console Workflow

- Submit every new page URL to Google Search Console after publishing.
- Confirm the page is included in `sitemap.xml` when the sitemap is updated later.
- Check indexing after a few days.
- After 8-10 planned pages are published, review:
  - Queries with impressions but low CTR
  - Pages discovered but not indexed
  - Keywords that suggest missing pages or FAQ sections
  - Internal pages receiving impressions but weak clicks
- Use Search Console findings to adjust titles, meta descriptions, intros, and internal links.
